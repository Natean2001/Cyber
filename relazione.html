Exploit XSS Persistente in Kimai 2

Questo è un esempio di un exploit che sfrutta una vulnerabilità di tipo Cross-Site Scripting (XSS) persistente in Kimai 2, una piattaforma di gestione del tempo open source.

Passo-passo dell'Exploit

  1. L'utente normale tenta di aggiungere un timesheet (registro delle ore lavorate) attraverso un link specifico nel software Kimai 2.
  2. Nella descrizione del timesheet, l'utente inserisce il payload ">
  3. L'utente salva le modifiche, inviando così il payload al server Kimai 2.
  4. Dopo aver salvato le modifiche, l'utente aggiorna la pagina. A questo punto, il payload viene eseguito perché il server Kimai 2 non ha sanificato correttamente i dati inseriti nella descrizione del timesheet. Di conseguenza, viene visualizzato un popup con il messaggio "xss", dimostrando che l'attacco XSS ha avuto successo.

Il payload è inserito all'interno del campo "timesheet_edit_form[description]" e viene interpretato come codice JavaScript dannoso quando visualizzato nel browser dell'utente. Questo è un esempio di XSS persistente, in cui il payload dannoso viene memorizzato nel server e poi restituito ad altri utenti quando visualizzano la pagina, consentendo all'attaccante di eseguire codice malevolo nel contesto del browser delle vittime.

Descrizione dell'exploit:

L'URL target dell'exploit è: http://host/patch/pagina.phtml?explode_tree=[SQL Injection]. L'obiettivo è il parametro explode_tree.

PoC (Proof of Concept):

L'exploit dimostra come sfruttare la SQL injection nell'URL. Utilizza una payload che sfrutta la funzione extractvalue per estrarre dati dal database MySQL.

Il payload '/*!50000and*/+/*!50000extractvalue*/(0x0a,/*!50000concat*/(0x0a,0x73337830753a,(/*!50000select*/ database()),0x3a7333783075))--+- viene utilizzato per eseguire una query SQL dannosa che estrae il nome del database MySQL corrente e lo concatena con altri dati. Il risultato verrà poi visualizzato nell'output HTTP.

La variante del payload utilizza la percentuale %20 per rappresentare uno spazio in URL-encoded e funziona in modo simile.
